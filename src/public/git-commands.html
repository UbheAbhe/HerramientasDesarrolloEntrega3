Comandos Esenciales de Git

Funcionalidad y uso de los comandos más importantes para tu flujo de trabajo.

1. git init

Crea un nuevo repositorio de Git en la carpeta actual. Es el primer paso para empezar a rastrear archivos.

2. git clone [URL]

Descarga un repositorio existente desde una URL remota (como GitHub) a tu máquina local.

3. git add [archivo] / git add .

Mueve los cambios de tu directorio de trabajo al **"área de staging"** (staging area), preparándolos para el commit.

4. git commit -m "[mensaje]"

Guarda los cambios preparados (staged) de forma permanente en el historial del repositorio local con un mensaje descriptivo.

5. git status

Muestra el estado actual del repositorio, incluyendo archivos modificados, archivos staged y archivos sin seguimiento.

6. git push origin [rama]

Sube los commits locales al repositorio remoto (GitHub, GitLab, etc.), compartiendo el historial con otros colaboradores.

7. git pull origin [rama]

Descarga los cambios del repositorio remoto y los fusiona (merge) automáticamente con tu rama local.

8. git branch

Muestra una lista de todas las ramas locales. Con un argumento, crea o cambia de nombre una rama.

9. git checkout [rama] / git switch [rama]

Cambia el directorio de trabajo a una rama o commit específico.

10. git merge [rama]

Integra los cambios de la rama especificada a la rama actual.

11. git log

Muestra el historial completo de commits del repositorio, permitiendo ver el ID del commit, el autor, la fecha y el mensaje. Es crucial para rastrear qué ha cambiado y cuándo.

12. git diff

Muestra las diferencias exactas entre el directorio de trabajo y el área de staging, entre dos commits, o entre dos ramas. Es la herramienta principal para revisar cambios antes de hacer un commit.

13. git stash

Guarda temporalmente todos los cambios en el directorio de trabajo y el área de staging sin la necesidad de hacer un commit formal. Esto es útil cuando necesitas cambiar de rama rápidamente pero no quieres comitear el trabajo incompleto.

14. git revert [commit ID]

Deshace un commit específico **creando un nuevo commit** que revierte los cambios. Este es el método más seguro para deshacer errores en un historial compartido, ya que no reescribe el historial.

15. git reset [archivo]

Se utiliza para deshacer commits o para sacar archivos del área de staging. Por defecto (modo `--mixed`), mueve los archivos staged de vuelta al directorio de trabajo.

16. git remote -v

Muestra una lista de las URLs de los repositorios remotos que Git tiene configurados (por ejemplo, `origin`).

17. git config --global user.name "Tu Nombre"

Define el nombre de usuario globalmente para Git, que se adjuntará a todos tus commits. Es un comando de configuración inicial.

18. git tag [nombre]

Crea una etiqueta permanente en un punto específico del historial, generalmente para marcar versiones de lanzamiento (ejemplo: `v1.0.0`).
